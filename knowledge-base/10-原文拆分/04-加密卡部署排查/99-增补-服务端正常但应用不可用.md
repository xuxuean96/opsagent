# 增补：服务端正常但应用不可用

## 场景

加密卡服务端已经启动成功，端口也正常监听，但 eCol 或 CLOUD 仍然无法使用授权功能。

## 优先怀疑

应用程序配置的加密卡服务地址不正确。

## 排查顺序

1. 确认 `KmPermitServer` 服务端进程正常
2. 确认端口监听正常，默认通常是 `5600`
3. 用独立客户端直连服务端验证授权可用
4. 检查 eCol/CLOUD 中配置的加密卡服务器 IP
5. 检查端口是否和服务端一致
6. 检查 CLOUD 客户端模式配置是否正确
7. 修改配置后重启 eCol/CLOUD 相关服务

## 典型结论

如果服务端和独立客户端都正常，只有 eCol/CLOUD 不可用，不要优先重装加密卡服务端，应先修正应用侧加密卡地址、端口和客户端模式配置。

## 补充分支：进程存在但客户机连不上

如果服务端进程已经存在，且服务端日志显示启动成功，但客户机、eCol 或 CLOUD 仍无法连接，需要优先检查防火墙和网络策略。

检查项：

1. 客户机是否能访问服务端 IP
2. 客户机是否能访问服务端端口，默认通常是 `5600`
3. 服务端防火墙是否放行该端口
4. 客户机防火墙或安全软件是否拦截
5. 跨网段时路由、ACL、安全组或网闸策略是否放行

## 补充分支：启动失败但已有进程

加密卡启动失败时，先检查是否已经存在 `KmPermitServer` 进程，或默认端口是否已经被占用。

处理建议：

- 已有正常进程时，不要重复启动
- 旧进程异常时，先停止旧进程再启动
- 端口被占用时，释放端口或修改服务端端口
- 重启后查看 `KmPermitServer.log` 判断是否真正启动成功
