# 01-附录DD：系统架构与组件关系（实施必须了解）

## 附录 DD：系统架构与组件关系（实施必须了解）

> 理解整体架构是排查问题的前提。本节说明各组件之间的调用关系和数据流向。

### DD.2 DDB 中间件详解

#### 核心定位
- **语言**：C++（VC++6/VS2015+，32位和64位版本）
- **性质**：基础服务 + 插件通信中间件
- **功能**：
  - 插件通信（COM接口）
  - 缓存数据存储/读取
  - CAD文件解析（DWG/ProE/CATIA/UG/SolidWorks/...）
  - 数据转换（XML/表缓存/轻量化）
  - 属性提取与写入
  - 签字与PDF转换

#### 核心组件

| 组件 | 文件 | 功能 | 注册方式 |
|------|------|------|----------|
| DDB调度包 | DDBDispatch.dll | 统一入口，加载配置，调用各驱动 | regsvr32 |
| DDB核心 | DDBCore.dll | 核心引擎，处理XML数据流 | regsvr32 |
| 通用数据处理 | DataPsr.dll | 通用数据解析，必须注册 | regsvr32 |
| DPL处理 | DDBDplPsr.dll | DPL脚本执行 | regsvr32 |
| AutoCAD驱动 | KmDwgBom.dll | DWG读写 | regsvr32 |
| ProE驱动 | DDBProeTranslator.dll | ProE/Creo读写 | regsvr32 |
| CATIA驱动 | CATIATranslator.dll | CATIA读写 | regsvr32 |
| UG驱动 | UgSeTranslator.dll | UG/SolidEdge读写 | regsvr32 |
| SolidWorks驱动 | SWTranslator.dll | SolidWorks读写 | regsvr32 |
| Inventor驱动 | InvTranslator.dll | Inventor读写 | regsvr32 |
| Excel驱动 | ExcelTranslator.dll | Excel导入 | regsvr32 |
| Word驱动 | WordTranslator.dll | Word导入/签字 | regsvr32 |
| CAXA驱动 | CAXATranslator.dll | CAXA读写 | regsvr32 |
| 中望驱动 | ZWTranslator.dll | 中望3D读写 | regsvr32 |
| 结构数据导入 | DDBNomalTrans.dll | Excel结构数据 | regsvr32 |
| 3DAST接口 | EBOMTranslator.dll | 轻量化转换 | regsvr32 |
| 表缓存接口 | EBOMDriver.dll | 表缓存输出 | regsvr32 |
| 日志组件 | KmDdbLog.dll / KmDdbLog_64.dll | 日志输出 | 无需注册 |
| DPL命令 | DpDplCom.exe | DPL脚本注册 | /RegServer |

#### 调用方式

**方式1：eCol 调用（C# → COM）**
```
eCol (C#)
  → 调用 COM 接口 (DDBDispatch.DDBPack)
  → DDBExec.exe 无界面模式
  → 加载 XML 配置
  → 调用对应 CAD 驱动 DLL
  → 生成 XML 或表缓存
```

**方式2：PLM-CLOUD 调用（Java → 命令行）**
```
PLM-CLOUD (Java)
  → 调用 DDBExec.exe 命令行
  → 传入参数：模式(OutXml/Update/Cable/Annot/Topdf) + 配置文件路径
  → DDBExec 加载配置
  → 调用驱动生成结果
  → 返回 XML 或表缓存文件
```

**方式3：测试桩直接调用**
```
DDBExec.exe \t          (GUI测试桩)
DDBExec.exe [段名] [ini路径]  (无界面命令行)
```

### DD.3 eCol 产品调用链

```
eCol 客户端 (C# .NET)
  │
  ├─ 导入功能
  │   → 调用 DDB COM 接口 (DDBDispatch)
  │   → 传入：CAD文件路径 + XML配置文件路径
  │   → 返回：XML数据或表缓存
  │   → eCol 解析 XML → 创建 PDM 对象
  │
  ├─ 签字功能
  │   → 调用 DDB COM 接口 (Update 模式)
 │   → 传入：update.xml + 签字内容
  │   → DDB 打开 CAD 文件 → 写入属性/图片
  │   → 保存文件
  │
  ├─ 转PDF功能
  │   → 调用 DDB COM 接口 (Topdf 模式)
  │   → 传入：CAD文件路径 + 配置
  │   → 生成 PDF 文件
  │
  └─ 轻量化功能
      → 调用 DDB (EBOMTranslator/EBOMDriver)
      → 调用 3DAST 转换引擎
      → 生成 .scs/.hsf 轻量化文件
```

**eCol 与 DDB 的关键交互点：**
- **COM 注册**：所有 DDB DLL 必须注册到系统注册表
- **配置文件路径**：通过环境变量 %kmsoftpath% 或绝对路径指定
- **日志位置**：DDBExec.exe 同目录下的 KmDdbLog.Txt
- **错误处理**：DDBError.txt 或 DDBError.log

### DD.4 PLM-CLOUD 产品调用链

```
PLM-CLOUD (Java Spring Boot)
  │
  ├─ 数据导入服务 (plm-data-import-service)
  │   → 接收前端上传的 CAD 文件
  │   → 调用 DDBExec.exe 命令行
  │   → 生成 XML 或表缓存
  │   → 解析结果 → 写入数据库
  │
  ├─ 文件批处理服务 (common-file-ext-service)
  │   → 接收转PDF/签字/轻量化任务
  │   → 调用 DDBExec.exe 命令行
  │   → 生成 PDF/签字文件/轻量化文件
  │   → 保存到文件柜
  │
  ├─ 加密卡验证 (Java 客户端)
  │   → 通过反射加载 KmEncryptCardClient.jar
  │   → 连接加密卡服务端 (5600/5700端口)
  │   → 验证授权 → 允许/拒绝操作
  │
  └─ KMVue 集成
      → 前端请求浏览轻量化文件
      → 调用 KMVue 服务 (Nginx + ODA)
      → 返回 .scs 或 PDF 文件流
```

**PLM-CLOUD 与 DDB 的关键交互点：**
- **命令行调用**：`DDBExec.exe [段名] [ini路径]`
- **配置文件**：通过 DDBExec.ini 指定各 CAD 的配置文件路径
- **输出目录**：表缓存文件输出到指定目录，Java 服务读取解析
- **错误日志**：Java 服务读取 DDBError.txt 获取错误信息

### DD.5 加密卡调用链（Java 反射方式）

```
PLM-CLOUD (Java)
  │
  ├─ 方式1：默认 Java 客户端 (mode=2)
  │   → 通过 URLClassLoader 反射加载 KmEncryptCardClient-1.0.3-fat.jar
  │   → 加载类：com.kmsoft.securitycard.service.EncryptionCardServiceEnhanced
  │   → 创建 ServiceConfig 配置对象
  │   → 调用方法：login / getAllProductInfo / getSingleProductInfo / logout
  │   → 返回 EncryptionCardResult 对象
  │
  └─ 方式2：原生客户端 (mode=1)
      → 调用 Windows/Linux 原生 C++ 客户端库
      → 通过 JNI 或命令行调用
      → 返回原生结构体数据
```

**Java 反射加载代码示例：**
```java
// 1. 动态加载 JAR
String jarPath = "./kmwp_lib/kmThirdGateway/ENCRYPTION/";
File[] jarFiles = new File(jarPath).listFiles((dir, name) ->
    name.startsWith("KmEncryptCardClient") && name.endsWith(".jar"));
URL[] urls = Arrays.stream(jarFiles).map(f -> f.toURI().toURL()).toArray(URL[]::new);
URLClassLoader classLoader = new URLClassLoader(urls, Thread.currentThread().getContextClassLoader());

// 2. 加载服务类
Class<?> serviceClass = classLoader.loadClass(
    "com.kmsoft.securitycard.service.EncryptionCardServiceEnhanced");
Class<?> configClass = classLoader.loadClass(
    "com.kmsoft.securitycard.service.EncryptionCardServiceEnhanced$ServiceConfig");

// 3. 创建配置
Object config = configClass.newInstance();
configClass.getMethod("setMaxRetries", int.class).invoke(config, 3);
configClass.getMethod("setRetryDelayMs", long.class).invoke(config, 1000L);
configClass.getMethod("setKeepAlive", boolean.class).invoke(config, true);

// 4. 创建服务实例
Object service = serviceClass.getConstructor(configClass).newInstance(config);

// 5. 调用方法
Method loginMethod = serviceClass.getMethod("login", String.class, int.class, int.class);
Object result = loginMethod.invoke(service, "192.168.1.100", 5600, 4);
```

**配置切换方式（wpConfig.yaml）：**
```yaml
kmwp.encryption.EncryptionCard.mode: 2  # 2=Java客户端(默认), 1=原生客户端
```

### DD.6 数据流向图

#### 导入流程
```
CAD文件 (DWG/PRT/ASM/...)
  ↓
DDBExec.exe + XML配置 (读写配置 + 处理配置)
  ↓
生成原始XML (bomData结构)
  ↓
处理配置 (属性对应/合并/DPL脚本/...)
  ↓
最终XML 或 表缓存 (14张表)
  ↓
eCol / PLM-CLOUD 解析
  ↓
创建PDM对象 (Part/Document/ChildPart/...)
  ↓
写入数据库
```

#### 签字流程
```
PLM系统触发签字
  ↓
生成 update.xml (签字内容 + 文件路径 + 配置路径)
  ↓
DDBExec.exe Update模式
  ↓
打开CAD文件
  ↓
写入属性/图片到指定坐标
  ↓
保存文件
  ↓
返回成功/失败
```

#### 转PDF流程
```
CAD文件
  ↓
DDBExec.exe Topdf模式
  ↓
调用CAD软件或第三方库 (AcadToPDF.exe/3DAST/...)
  ↓
生成PDF文件
  ↓
保存到文件柜
```

#### 轻量化流程
```
3D模型文件 (CATIA/UG/ProE/...)
  ↓
DDBExec.exe + EBOMTranslator/EBOMDriver
  ↓
调用3DAST转换引擎 (ExternConvert.exe)
  ↓
生成轻量化文件 (.scs/.hsf)
  ↓
生成XML结构数据
  ↓
PLM-CLOUD 导入数据库
  ↓
KMVue 浏览轻量化文件
```

### DD.8 常见问题与架构关系

| 问题现象 | 可能涉及的层 | 排查方向 |
|----------|-------------|----------|
| 导入失败 | DDB层/CAD驱动层 | 组件注册、环境变量、CAD软件安装 |
| 签字失败 | DDB层/配置层 | update.xml格式、坐标配置、CAD文件权限 |
| 转PDF失败 | DDB层/第三方工具层 | 字体缺失、打印样式、CAD软件安装 |
| 轻量化失败 | DDB层/3DAST层 | 3DAST安装、环境变量、加密软件冲突 |
| 加密卡连不上 | Java层/网络层/服务端 | 端口、防火墙、mode配置、授权文件 |
| 表缓存为空 | DDB层/配置层 | 配置文件编码、路径大小写(Linux)、DPL脚本 |
| 日志乱码 | DDB层/编码层 | 配置文件编码(GB2312)、系统区域设置 |
| 组件加载失败 | 系统层/DDB层 | DLL注册、依赖缺失、32/64位不匹配 |

### DD.9 32位 vs 64位注意事项

| 组件 | 32位 | 64位 | 说明 |
|------|------|------|------|
| DDBExec.exe | ✅ | ✅ | 测试桩，有GUI |
| DDBExec_64.exe | ❌ | ✅ | 64位测试桩 |
| eCol | ✅ | ❌ | C#客户端，32位 |
| PLM-CLOUD | N/A | ✅ | Java，自动适配 |
| AutoCAD | ✅ | ✅ | DDB仅用32位 |
| ProE/Creo | ✅ | ✅ | DDB仅用32位 |
| CATIA | ✅ | ✅ | DDB支持两者 |
| UG | ✅ | ✅ | DDB支持两者 |
| SolidWorks | ✅ | ✅ | DDB用32位 |
| Inventor | ✅ | ✅ | DDB用32位 |
| CAXA | ✅ | ❌ | DDB仅用32位 |
| 中望3D | ✅ | ✅ | 2023+支持64位 |
| Excel/Word | ✅ | ✅ | DDB用32位 |
| 3DAST | ❌ | ✅ | 仅64位 |
| KmDdbLog.dll | ✅ | ❌ | 32位日志 |
| KmDdbLog_64.dll | ❌ | ✅ | 64位日志 |

**关键规则：**
- eCol 是 32 位，调用 32 位 DDB 组件
- PLM-CLOUD 是 Java，通过命令行调用，32/64 自动适配
- 3DAST 是 64 位，必须用 64 位 DDB 调用
- 多数 CAD 驱动有 32 位和 64 位版本，但 DDB 框架通常使用 32 位

### DD.10 部署架构要点

#### eCol 部署
```
客户端机器
  ├─ eCol 安装 (C# .NET Framework)
  ├─ 公共组件安装 (D:\KMSOFT\COMMON)
  │   ├─ DDB 组件 (DLL + EXE)
  │   ├─ C平台 (C++运行时)
  │   └─ 各CAD驱动
  ├─ CAD软件 (AutoCAD/UG/CATIA/...)
  └─ 环境变量配置 (%kmsoftpath%, %UGII_ROOT_DIR%, ...)
```

#### PLM-CLOUD 部署
```
应用服务器
  ├─ Java 运行时 (JDK 8+)
  ├─ Tomcat/东方通/宝兰德 (中间件)
  ├─ PLM-CLOUD 服务 (多个 jar)
  │   ├─ plm-service (业务服务)
  │   ├─ plm-data-import-service (数据导入)
  │   ├─ common-file-ext-service (文件批处理)
  │   └─ ...
  ├─ 公共组件 (D:\KMSOFT\COMMON)
  │   └─ DDB 组件
  ├─ 文件柜 (文件存储目录)
  └─ 配置文件 (run/config/wpConfig.yaml)

数据库服务器
  └─ Oracle/DM/MySQL (PDM数据库)

加密卡服务器 (可选)
  ├─ Windows: KmPermitServer.exe (5600端口)
  └─ Linux: KmPermitServer_Linux (5700端口)

KMVue服务器 (可选)
  ├─ Nginx (前端)
  ├─ ODA服务 (DWG浏览)
  └─ 转换服务 (SCS生成)
```
