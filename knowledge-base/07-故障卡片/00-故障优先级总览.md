# 故障优先级总览

## 通用排障顺序

1. 确认现象和报错全文
2. 确认组件、系统、位数、调用方式
3. 服务无法启动时，先查运行环境和依赖
4. 查进程和端口
5. 查日志第一条错误
6. 查配置文件和路径
7. 查权限和防火墙
8. 查版本、注册和依赖
9. 最后考虑重装或覆盖组件

## 启动失败优先级

1. 直接执行程序，确认能否独立启动
2. 查 Java、运行库、`.so`、`GLIBCXX`、`libstdc++` 等运行时
3. 查环境变量，例如 PATH、`LD_LIBRARY_PATH`
4. 查进程和端口
5. 查业务配置

特别注意：KMVue Linux 报加密卡连接不上时，也要检查 `kmconvertservice` 是否缺少运行时，不能只按加密卡连接问题处理。

## 加密卡优先级

1. `KmPermitServer` 进程是否存在
2. 端口是否监听，默认通常是 `5600`
3. 客户机是否能访问服务端端口
4. 服务端和客户机防火墙是否放行
5. eCol/CLOUD 配置的加密卡 IP 和端口是否正确
6. CLOUD 客户端模式是否正确
7. 授权文件是否存在、有效、未过期
8. 授权节点是否已满
9. 硬件、驱动、依赖库是否异常

## DDB 优先级

1. DDBExec 是否能启动
2. 公共组件目录是否完整
3. 32/64 位是否匹配
4. COM 组件是否注册
5. XML 配置路径和编码是否正确
6. 输入文件是否存在且可访问
7. 输出目录是否可写
8. `KmDdbLog.Txt` 和 `DDBError.log` 中第一条错误
