# 加密卡-服务启动但应用不可用

## 适用场景

加密卡服务端已经启动，独立客户端能连通，但 eCol 或 CLOUD 仍提示授权失败、加密卡不可用或许可不可用。

## 典型现象

- `KmPermitServer` 进程存在
- 端口已监听
- 独立客户端测试成功
- eCol/CLOUD 内部功能仍不可用

## 可能原因

| 优先级 | 原因 | 判断依据 |
|---|---|---|
| P1 | 应用配置的加密卡 IP 不对 | 服务端正常，但应用仍连旧地址 |
| P1 | 应用配置的端口不对 | 服务端端口和配置端口不一致 |
| P2 | CLOUD 客户端模式不匹配 | `kmwp.encryption.EncryptionCard.mode` 设置不对 |
| P2 | 修改配置后未重启应用 | 配置文件已改但应用仍用旧配置 |

## 先问用户

1. 独立客户端能否连上加密卡服务端
2. eCol/CLOUD 配置的加密卡 IP 和端口是多少
3. 服务端实际 IP 和端口是多少
4. CLOUD 的 `kmwp.encryption.EncryptionCard.mode` 是多少

## 检查步骤

1. 确认服务端进程和端口正常
2. 用独立客户端直连服务端
3. 检查 eCol/CLOUD 配置中的服务端 IP
4. 检查端口，默认通常是 `5600`
5. 检查 CLOUD 客户端模式
6. 修改配置后重启应用服务

## 验证方式

- eCol/CLOUD 授权功能恢复
- 应用日志不再报加密卡连接失败
- `KmPermitServer.log` 能看到应用侧连接记录
