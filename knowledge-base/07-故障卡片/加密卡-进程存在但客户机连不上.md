# 加密卡-进程存在但客户机连不上

## 适用场景

服务端机器上 `KmPermitServer` 进程存在，日志显示启动成功，但客户机、eCol 或 CLOUD 连接失败。

## 可能原因

| 优先级 | 原因 | 判断依据 |
|---|---|---|
| P1 | 服务端防火墙未放行端口 | 服务端本机能连，客户机不能连 |
| P1 | 客户机防火墙或安全软件拦截 | 部分机器能连，部分机器不能连 |
| P1 | 网络策略阻断 | 跨网段、ACL、安全组或网闸未放行 |
| P2 | 服务监听 IP 不对 | 监听在本机地址或错误网卡 |

## 先问用户

1. 客户机能否 ping 通服务端
2. 客户机能否访问服务端 `5600` 端口
3. 服务端防火墙是否放行该端口
4. 是否跨网段或有安全设备

## 检查步骤

1. 服务端确认 `KmPermitServer` 进程存在
2. 服务端确认端口监听
3. 客户机测试端口连通性
4. 检查服务端入站防火墙
5. 检查客户机出站防火墙和安全软件
6. 跨网段时检查网络策略

## 验证方式

- 客户机能连通服务端端口
- eCol/CLOUD 不再报连接失败
- 服务端日志出现客户机连接记录
