# 加密卡-启动失败已有进程

## 适用场景

启动加密卡服务端失败，但机器上可能已经存在 `KmPermitServer` 进程或端口已被占用。

## 可能原因

| 优先级 | 原因 | 判断依据 |
|---|---|---|
| P1 | 服务已经启动 | 已有 `KmPermitServer` 进程 |
| P1 | 端口已被占用 | 默认端口 `5600` 被占用 |
| P2 | 旧进程僵死 | 进程存在但不响应 |
| P2 | 重复启动多个实例 | 多个启动脚本或服务同时运行 |

## 先问用户

1. 当前是否已有 `KmPermitServer` 进程
2. `5600` 端口是否已监听
3. 日志最后一条错误是什么

## 检查步骤

1. 查询 `KmPermitServer` 进程
2. 查询端口占用
3. 判断进程是否正常响应
4. 如果旧进程异常，先停止旧进程
5. 再重新启动服务端

## 验证方式

- 只有一个有效服务端进程
- 端口正常监听
- 客户端可以连接
