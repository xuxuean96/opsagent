# DDB-组件加载失败

## 适用场景

日志或界面提示不能加载 DDB 组件、找不到 DLL、未找到注册类或组件不兼容。

## 可能原因

| 优先级 | 原因 | 判断依据 |
|---|---|---|
| P1 | 组件未注册 | 报未找到注册类或 COM 创建失败 |
| P1 | 32/64 位混用 | `regsvr32` 报不兼容 |
| P1 | 依赖 DLL 缺失 | 组件加载失败但文件存在 |
| P2 | 公共组件版本混乱 | 部分覆盖后多个功能异常 |

## 先问用户

1. 报错组件名是什么
2. 用的是 32 位还是 64 位
3. 是否最近覆盖过 common 目录
4. 是否管理员权限注册过组件

## 检查步骤

1. 确认组件文件存在
2. 用对应位数的 `regsvr32` 注册
3. 检查依赖 DLL
4. 确认 common 目录整套覆盖
5. 重新运行 DDBExec 验证

## 验证方式

- 组件能正常加载
- 不再报未注册或不兼容
- `KmDdbLog.Txt` 中进入后续处理流程
