# 通用反问策略

## 只问最少必要信息

用户信息不足时，最多先问 3-5 个关键问题，不要一次性要求全部材料。

## 通用必问信息

1. 组件名或功能名
2. 操作系统和位数
3. 调用方式，GUI、CLI、eCol、CLOUD 或服务调用
4. 报错全文
5. 日志第一条错误
6. 服务无法启动时，直接执行程序的报错是什么

## 什么时候直接给方案

如果用户已经给出明确报错关键词，可以先给最可能原因和前三步检查。
如果缺少关键证据，再让用户补充日志。

## 什么时候必须追问

- 没有组件名
- 没有报错全文
- 不知道是服务端还是客户端
- 不知道调用方式
- 现象可能对应多个完全不同方向

## 服务无法启动时必须追问

1. 是哪个服务无法启动
2. 是否能在命令行直接执行该程序
3. 是否报 Java、`.so`、`GLIBCXX`、`libstdc++` 等运行时错误
4. 是否配置了 PATH 或 `LD_LIBRARY_PATH`
5. Linux 上是否直接执行过 DDBExec、kmconvertservice 或 KmPermitServer_Linux
