# 加密卡反问策略

## 用户说“加密卡连不上”

先问：

1. 服务端 `KmPermitServer` 进程在吗
2. 服务端端口 `5600` 监听了吗
3. 客户机能访问服务端 `5600` 吗
4. 是独立客户端连不上，还是 eCol/CLOUD 连不上
5. 服务端日志第一条错误是什么

## 用户说“服务启动了但系统不能用”

先问：

1. 独立客户端能不能连上服务端
2. eCol/CLOUD 配置的加密卡 IP 是多少
3. 端口是否和服务端一致
4. CLOUD 的 `kmwp.encryption.EncryptionCard.mode` 是多少

## 用户说“启动失败”

先问：

1. 是否已有 `KmPermitServer` 进程
2. `5600` 端口是否被占用
3. `KmPermitServer.log` 最后一条错误是什么
4. 是 Windows 服务端还是 Linux 服务端
5. Linux 上是否直接执行过 `KmPermitServer_Linux_Server.sh` 或 `KmPermitServer_Linux`
6. 是否有缺少 `.so`、`GLIBCXX`、`libstdc++` 的错误

## 用户说“有的机器能连，有的不能”

优先怀疑客户机侧防火墙、安全软件、网段策略。
先问不能连接的机器和能连接的机器是否在同一网段、是否有相同安全策略。
