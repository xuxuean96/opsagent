# 知识库治理策略

## 目标

知识库必须服务于运维问答 Agent 的稳定回答，治理目标包括：

1. 提高检索命中率。
2. 降低重复、过期、低质量内容对回答的干扰。
3. 保证故障卡片、FAQ、反问策略、测试集的边界清晰。
4. 让每次知识库变更都可以被自动检查和复盘。

## 文档状态

| 状态 | 含义 | 检索策略 |
|---|---|---|
| `active` | 当前可用内容 | 正常检索 |
| `draft` | 草稿，未验证 | 默认可以检查，但上线前应转为 `active` |
| `deprecated` | 已废弃或被替代 | 应降低权重或从生产索引中排除 |

## 元数据要求

每个正式文档建议包含 YAML front matter：

```yaml
---
type: runbook
component: 加密卡
scenario: 启动失败
priority: P1
tags: [加密卡, 启动失败, KmPermitServer]
status: active
---
```

必填字段：

| 字段 | 说明 |
|---|---|
| `type` | 文档类型，例如 `runbook`、`faq`、`reference`、`strategy`、`testcase`、`standard` |
| `component` | 组件名称 |
| `scenario` | 问题场景 |
| `priority` | 优先级，允许 `P0`、`P1`、`P2`、`P3` |
| `tags` | 检索标签，必须是非空列表 |
| `status` | 文档状态，允许 `active`、`draft`、`deprecated` |

## 目录与类型对应

| 目录 | 推荐类型 |
|---|---|
| `07-故障卡片` | `runbook` |

## 质量门禁

每次修改知识库后执行：

```powershell
python -m ops_agent.cli govern --report data/governance-report.md
```

上线前建议执行严格模式：

```powershell
python -m ops_agent.cli govern --strict --report data/governance-report.md
```

治理检查项：

1. 是否缺少 YAML front matter。
2. 是否缺少必填元数据。
3. `type`、`priority`、`status`、`tags` 是否合法。
4. 目录推荐类型和文档 `type` 是否一致。
5. 正文是否缺少一级标题。
6. 正文是否过短。
7. 是否存在重复内容。
8. 是否存在废弃文档。

## 变更流程

1. 新增或修改知识文档。
2. 执行治理检查。
3. 执行索引重建。
4. 使用测试集或典型问题验证检索来源。
5. 根据日志和用户反馈补充故障卡片或反问策略。

## 内容编写原则

1. 故障卡片优先写“判断依据”和“处理步骤”，不要只写结论。
2. FAQ 适合标准短问答，不承担复杂排障流程。
3. 反问策略只写信息收集逻辑，不替代故障卡片。
4. 原文拆分可以保留完整背景，但高频问题必须沉淀到故障卡片。
5. 每条命令都应说明适用系统、风险和预期结果。
