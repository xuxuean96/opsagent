# 元数据规范

## 必填字段

| 字段 | 含义 | 示例 |
|---|---|---|
| `type` | 文档类型 | `runbook`、`faq`、`reference`、`testcase` |
| `component` | 组件 | `加密卡`、`DDB`、`Creo`、`Excel` |
| `scenario` | 场景 | `客户端连接失败`、`生成XML失败` |
| `priority` | 优先级 | `P0`、`P1`、`P2`、`P3` |
| `tags` | 检索标签 | `[加密卡, CLOUD, 防火墙]` |
| `status` | 状态 | `active`、`draft`、`deprecated` |

## 优先级定义

| 优先级 | 含义 |
|---|---|
| `P0` | 阻断生产或核心业务不可用 |
| `P1` | 高频故障或强影响问题 |
| `P2` | 中频问题，有明确绕过方式 |
| `P3` | 低频问题、解释性问题或补充知识 |

## 文档类型

| 类型 | 用途 |
|---|---|
| `runbook` | 故障卡片，指导排障 |
| `faq` | 标准问答 |
| `reference` | 参数、路径、术语、映射 |
| `strategy` | 反问策略、回答策略 |
| `testcase` | 测试集 |

## 标签建议

标签应覆盖：

- 组件名：`加密卡`、`DDB`、`Creo`
- 产品名：`eCol`、`CLOUD`
- 现象：`启动失败`、`连不上`、`生成XML失败`
- 关键证据：`防火墙`、`端口5600`、`KmPermitServer`
- 处理方向：`配置IP`、`注册COM`、`环境变量`
