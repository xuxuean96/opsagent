# 反问测试集

## 测试目标

验证机器人在信息不足时是否能问出最关键的问题。

## 测试用例

| ID | 用户问题 | 期望反问 |
|---|---|---|
| C001 | 加密卡连不上 | 进程在吗、端口监听了吗、客户机能访问端口吗、是独立客户端还是 eCol/CLOUD |
| C002 | DDB 报错 | 数据源类型、操作模式、GUI/CLI、日志第一条错误 |
| C003 | 生成 XML 失败 | DDBExec.ini 段名、ConfigFile、输入路径、输出路径、日志 |
| C004 | 签字失败 | Word/Excel/CAD、文字/图片签字、报错全文、Office 是否正常 |
| C005 | 转 PDF 失败 | 组件类型、输入文件、Office/CAD 是否安装、日志、输出目录 |
| C006 | CLOUD 不能用 | 哪个功能不可用、应用日志、是否修改过配置、服务是否重启 |
| C007 | 组件加载失败 | 具体 DLL、位数、是否注册、是否覆盖 common |
| C008 | 服务启动失败 | 哪个服务、操作系统、直接执行程序报错、是否缺运行时、环境变量 |
| C009 | KMVue 报加密卡连不上 | kmconvertservice 是否能独立启动、是否缺运行时、再查加密卡 |
| C010 | Linux DDBExec 启动不了 | 直接执行 DDBExec 的报错、so/GLIBCXX/libstdc++、LD_LIBRARY_PATH |
